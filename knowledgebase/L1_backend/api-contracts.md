# L1 — API Contracts

> Forward-mode contract generated by `backend-architect` on 2026-05-15.
> All endpoints are `Status: proposed`.
> Every endpoint cites the User Story IDs it serves.
> Base URL: `/api/v1` — all paths below are relative to this prefix.
> Auth scheme: Keycloak JWT passed as `Authorization: Bearer <token>`.
> Engine endpoints are namespaced under `/engine/*` and require a Keycloak **service account** JWT with the `engine` role.

---

## Response envelope

All endpoints return JSON. Success and error shapes follow the platform convention:

```json
// Success (2xx)
{
  "data": { ... } | [ ... ],
  "meta": { "page": 1, "limit": 20, "total": 42 }  // pagination only when applicable
}

// Error (4xx / 5xx)
{
  "error": {
    "code": "DOMAIN_NOT_FOUND",
    "message": "Domain with clientKey 'foo' not found",
    "field": "clientKey"  // optional — present for field-level errors
  }
}
```

## Auth: Keycloak JWT validation

> serves: US-001, US-002

All endpoints in `/api/v1/*` except health/probe are protected by a NestJS `JwtAuthGuard` that validates the Keycloak JWT signature, expiry, and realm. A `RolesGuard` enforces role-based access (`analyst` | `admin`) on tagged endpoints. Engine endpoints use a separate `EngineJwtGuard` that checks the `engine` role claim on the service account JWT.

**No dedicated login endpoint exists on this BE.** The FE handles the OIDC Authorization Code Flow directly with Keycloak. The BE is a resource server only — it validates Bearer tokens.

## Domain management (US-003, US-004, US-005, US-006)

### E-002 — Create domain

- **Method + path:** `POST /domains`
- **Auth:** Bearer JWT — role: `analyst` or `admin`
- **Summary:** Creates a new Domain entry. The caller's `userId` is automatically associated to the new `clientKey` in `userprofiles` (or the admin must pre-create the userprofile). `clientKey` is immutable after creation.
- **Request body:**
  ```json
  {
    "clientKey": "caffevergnano",
    "targetDomain": "caffevergnano.com",
    "brand": "Caffè Vergnano",
    "aliases": ["Vergnano", "1882 Caffè Vergnano"],
    "settori": ["food-beverage", "coffee"]
  }
  ```
  - `clientKey` required, string, slug-safe (`[a-z0-9-_]+`)
  - `targetDomain` required, string
  - `brand` required, string
  - `aliases` optional, string[]
  - `settori` optional, string[]
- **Response `201`:**
  ```json
  {
    "data": {
      "_id": "<mongo-objectid>",
      "clientKey": "caffevergnano",
      "targetDomain": "caffevergnano.com",
      "brand": "Caffè Vergnano",
      "aliases": ["Vergnano"],
      "settori": ["food-beverage"],
      "createdAt": "2026-05-15T10:00:00.000Z"
    }
  }
  ```
- **Errors:**
  - `401 Unauthorized` — missing or expired JWT
  - `403 Forbidden` — token has no valid role
  - `409 Conflict` — `clientKey` already exists (`error.code: DOMAIN_DUPLICATE_KEY`, `error.field: clientKey`)
  - `422 Unprocessable Entity` — `clientKey`, `targetDomain`, or `brand` missing or format invalid
  - `500 Internal Server Error` — MongoDB write failure
- **serves:** US-003 (AC-005, AC-006)

## Run lifecycle — user-facing (US-007, US-008, US-009, US-010, US-011, US-012, US-026)

### E-006 — Create and start run

- **Method + path:** `POST /domains/:clientKey/runs`
- **Auth:** Bearer JWT — role: `analyst` or `admin`
- **Summary:** Creates a new Run for the given domain. If no run is currently `running` for this domain, status is set to `running` and the n8n webhook is triggered synchronously within 5 seconds. If a run is already `running`, the new run is created in `queued` state and no webhook is sent. On successful n8n trigger, `executionId` from n8n's response is persisted in `georuns`.
- **Request body:**
  ```json
  {
    "profileKey": "openai-gpt4o-standard",
    "runIterations": 3,
    "locales": ["it-IT", "en-US"],
    "keywordsOverride": ["caffè specialty", "espresso italiano"],
    "testMode": false,
    "debugMode": false
  }
  ```
  - `profileKey` required — must exist in `profiles` collection
  - `runIterations` required — positive integer
  - `locales` optional — defaults to profile/registry default
  - `keywordsOverride` optional — overrides geo-registry base keywords
  - `testMode` optional, bool, default false
  - `debugMode` optional, bool, default false
- **Response `201`:**
  ```json
  {
    "data": {
      "runId": "2026-05-15T10:00:00.000Z_caffevergnano.com",
      "clientKey": "caffevergnano",
      "profileKey": "openai-gpt4o-standard",
      "status": "running",
      "runIterations": 3,
      "locales": ["it-IT", "en-US"],
      "keywordsOverride": ["caffè specialty"],
      "testMode": false,
      "debugMode": false,
      "executionId": "abc123",
      "plannedQueries": 42,
      "doneQueries": 0,
      "errorQueries": 0,
      "startedAt": "2026-05-15T10:00:00.000Z",
      "createdAt": "2026-05-15T10:00:00.000Z"
    }
  }
  ```
  When queued, `status` is `"queued"` and `executionId` is `null`.
- **Errors:**
  - `401 Unauthorized`
  - `403 Forbidden`
  - `404 Not Found` — `clientKey` not found or out of scope; `profileKey` not found
  - `422 Unprocessable Entity` — `profileKey` or `runIterations` missing; `runIterations` not positive integer
  - `502 Bad Gateway` — n8n webhook unreachable; run is created with `status: error` and `errorMessage` persisted; body includes `error.code: N8N_TRIGGER_FAILED`
  - `500 Internal Server Error`
- **serves:** US-007 (AC-011, AC-012), US-008 (AC-013, AC-014, AC-015), US-026 (AC-040 — indirect: queue logic executes at creation time)

> OQ-013 resolved (2026-05-15): `plannedQueries` computed by BE at run creation — **Opzione A**. Formula: `effectiveKeywords × locales.length × 7`. `effectiveKeywords` = `keywordsOverride.length` if provided, else platform default (10). `7` = fixed persona count per geo-registry convention. Applied synchronously before the n8n webhook trigger.

### E-009 — Cancel run

- **Method + path:** `DELETE /domains/:clientKey/runs/:runId`
- **Auth:** Bearer JWT — role: `analyst` or `admin`
- **Summary:** Cancels a run in `running` state. BE calls `DELETE /api/v1/executions/:executionId` on the n8n API. On n8n success, `georuns.status` is set to `cancelled` and `completedAt` is stamped. After cancellation, the queue promotion logic (US-026) runs automatically. Cancelling a `queued` run sets it directly to `cancelled` without contacting n8n.
- **Request body:** none
- **Response `200`:**
  ```json
  {
    "data": {
      "runId": "2026-05-15T10:00:00.000Z_caffevergnano.com",
      "status": "cancelled",
      "completedAt": "2026-05-15T11:30:00.000Z"
    }
  }
  ```
- **Errors:**
  - `401 Unauthorized`
  - `403 Forbidden`
  - `404 Not Found` — runId not found or out of scope
  - `409 Conflict` — run is not in `running` or `queued` state (already `done`, `error`, or `cancelled`) — `error.code: RUN_NOT_CANCELLABLE`
  - `502 Bad Gateway` — n8n DELETE call failed; run state NOT changed; `error.code: N8N_CANCEL_FAILED`
  - `500 Internal Server Error`
- **serves:** US-010 (AC-042, AC-043, AC-044, AC-045)

> OQ-014 resolved (2026-05-15): Cancelling a `queued` run **is allowed**. E-009 branches internally: `running` → n8n DELETE + `cancelled`; `queued` → direct `cancelled` (no n8n call, no `executionId` needed). AC-045 added to cover the queued case. SC-021 and SC-022 show "Annulla" button for both `running` and `queued` states.

### E-018 — Delete LLM profile

- **Method + path:** `DELETE /profiles/:profileKey`
- **Auth:** Bearer JWT — role: `admin` only

### E-022 — Admin: update user profile

- **Method + path:** `PATCH /admin/users/:userId`
- **Auth:** Bearer JWT — role: `admin` only
- **Summary:** Updates a user's `clientKeys`, `role`, or `active` flag. When `active: false` is set, the BE performs three sequential actions: (1) sets `userprofiles.active = false` in MongoDB; (2) calls Keycloak Admin API `PUT /admin/realms/{realm}/users/{id}` with `{ enabled: false }` to disable the KC account (blocks new token issuance); (3) calls Keycloak Admin API `DELETE /admin/realms/{realm}/users/{id}/sessions` to revoke all active sessions (invalidates already-issued tokens). Zero JWT TTL window exposure (OQ-018 Opzione C resolved).
- **Request body (all optional):**
  ```json
  {
    "clientKeys": ["caffevergnano", "lavazza"],
    "role": "analyst",
    "active": false
  }
  ```
- **Response `200`:** updated `userprofile` document
- **Errors:**
  - `401 Unauthorized`
  - `403 Forbidden`
  - `404 Not Found` — `userId` not found
  - `422 Unprocessable Entity`
  - `500 Internal Server Error`
- **serves:** US-023 (AC-037)

> OQ-018 resolved (2026-05-15): **Opzione C — revoca immediata completa**. On `active: false`, the BE calls: (1) MongoDB `userprofiles.active = false`; (2) Keycloak Admin API `PUT /users/{id}` `{ enabled: false }` — blocks new token issuance; (3) Keycloak Admin API `DELETE /users/{id}/sessions` — revokes existing tokens immediately. Zero JWT TTL window. See I-001 for Admin API credentials.

## Engine API — n8n → BE (US-019, US-020, US-021, US-022, US-026)

> All endpoints in `/engine/*` require Keycloak **service account** JWT with role `engine`.
> Rejections from missing/expired/wrong-role token → `401 Unauthorized` (AC-030).
> No FE or human user should call these endpoints.

### E-023 — Engine: patch run status

- **Method + path:** `PATCH /engine/runs/:runId`
- **Auth:** Engine JWT (service account, role: `engine`)
- **Summary:** n8n calls this to update the run status to `done` or `error`, set final counters, and optionally report the Google Drive `reportFolderId` and report file URLs. After this call, if status transitions to `done | error`, the BE automatically runs queue promotion logic (US-026 AC-040, AC-041). n8n cannot set status to `cancelled` (AC-044 — 409 if run is already `cancelled`).
- **Request body:**
  ```json
  {
    "status": "done",
    "plannedQueries": 42,
    "doneQueries": 41,
    "errorQueries": 1,
    "reportFolderId": "1AbCdEfGhIjKlMnOpQrStUvWx",
    "reportFiles": [
      { "type": "xlsx", "name": "report_caffevergnano_2026-05-15.xlsx", "driveUrl": "https://drive.google.com/..." },
      { "type": "md", "name": "report_caffevergnano_2026-05-15.md", "driveUrl": "https://drive.google.com/..." }
    ],
    "errorMessage": null
  }
  ```
  - `status` required: `done | error`
  - `plannedQueries`, `doneQueries`, `errorQueries` required for `done`; optional for `error`
  - `reportFolderId` and `reportFiles` optional — present only on `done` with successful Drive upload (OQ-016 resolved: URLs stored here, not fetched live)
  - `rankingSnapshot` optional — present only on `done`; array of brand objects pre-computed by n8n Finalizer; consumed directly by E-011 (OQ-015 resolved)
  - `errorMessage` required if `status: error`
- **Response `200`:**
  ```json
  {
    "data": {
      "runId": "...",
      "status": "done",
      "queuePromoted": true,
      "promotedRunId": "2026-05-14T09:00:00.000Z_caffevergnano.com"
    }
  }
  ```
  `queuePromoted` is `false` if no queued run existed for the domain.
- **Errors:**
  - `401 Unauthorized` — invalid or missing engine JWT (AC-030)
  - `404 Not Found` — `runId` not found in `georuns`
  - `409 Conflict` — run is already in terminal state (`cancelled`, `done`, `error`) — `error.code: RUN_TERMINAL_STATE` (AC-044)
  - `422 Unprocessable Entity` — `status` not in allowed set; `errorMessage` missing when `status: error`
  - `500 Internal Server Error`
- **serves:** US-022 (AC-035, AC-036), US-026 (AC-040, AC-041)

### E-027 — Engine: create LLM response (raw)

- **Method + path:** `POST /engine/queries/:queryId/llm-responses`
- **Auth:** Engine JWT (service account, role: `engine`)
- **Summary:** n8n calls this once per LLM iteration to persist the raw response before NER extraction. Must be called **before** E-026 for the same iteration — E-026 requires a valid `responseId` generated here. Resolves OQ-019: preserves the invariant that n8n never writes to MongoDB directly.
- **Request body:**
  ```json
  {
    "runId": "2026-05-15T10:00:00.000Z_caffevergnano.com",
    "modelName": "gpt-5-mini",
    "iterationIdx": 1,
    "promptText": "Quali sono i migliori caffè in grani italiani?",
    "systemPrompt": "Sei un assistente AI locale basato in Italia...",
    "llmResponse": "I migliori caffè in grani italiani includono Illy, Lavazza..."
  }
  ```
  - `runId` required — must exist in `georuns`
  - `modelName` required
  - `iterationIdx` required int ≥ 1
  - `promptText` required
  - `systemPrompt` required
  - `llmResponse` required
- **Response `201`:**
  ```json
  {
    "data": {
      "responseId": "2026-05-15T10:00:00.000Z_caffevergnano.com-<queryId>-1-<hash>",
      "queryId": "...",
      "runId": "...",
      "iterationIdx": 1
    }
  }
  ```
  The returned `responseId` must be passed as input to the subsequent E-026 call for this iteration.
- **Errors:**
  - `401 Unauthorized` (AC-030)
  - `404 Not Found` — `runId` or `queryId` (from path) not found
  - `409 Conflict` — a response with the same `queryId` + `iterationIdx` already exists (`error.code: RESPONSE_DUPLICATE_ITERATION`)
  - `422 Unprocessable Entity` — required fields missing
  - `500 Internal Server Error`
- **serves:** US-021 (prerequisite step before AC-033)
